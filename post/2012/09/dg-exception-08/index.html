解决方法

将报错的文件，按下面命令在DG端创建，然后再起MRP进程即可。

1
2
3
4

1. alter system set standby_file_management=manual;
2. alter database create datafile 25 as '/path/xt/trustdw/data/oradata/trustdw/irpdata03.dbf';
3. alter system set standby_file_management=auto;
4. alter database recover managed standby database disconnect parallel 8;

分析总结

本次异常，是由于在DG同步异常期间，生产添加了新的数据文件，同事在处理异常时，在生产生成新的standby的控制文件，从而导致控制文件认为该存在，但是实际上OS层面没有该文件，因此无法识别文件。

解决方法就是在OS层创建该文件，有了文件，MRP进程就可以对该文件进程恢复。

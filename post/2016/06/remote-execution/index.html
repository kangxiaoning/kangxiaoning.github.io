3. Master端返回结果到Mysql

这种方式只需要在master端配置和重启服务，易于操作，并且DB连接信息只需要保存在master端，复杂度和安全比较可控。

**原理：**master会把minion返回的结果缓存在/var/cache/salt/master/jobs中，所以有条件将结果保存到指定存储位置。

**注意：**实验结果表明启用master返回结果到mysql后，/var/cache/salt/master/jobs将不再保存执行结果。

a) master_job_cache

在master配置中设置master_job_cache后，命令执行结果不会缓存到本地，而是返回到由这个配置指定的位置。

ext_job_cache

Default: '' Used to specify a default returner for all minions. When this option is set, the specified returner needs to be properly configured and the minions will always default to sending returns to this returner. is will also disable the local job cache on the master.

master_job_cache

New in version 2014.7.0. Default: local_cache Specify the returner to use for the job cache.The job cache will only be interacted with from the salt master and therefore does not need to be accessible from the minions. this will make the Master store the job data using a returner (instead of the local job cache on disk).

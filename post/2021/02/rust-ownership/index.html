Rust Ownership

Ownership表达的是一种关系，在人类社会中的ownership表示owner对其拥有的资源享有所有权，比如修改、借用、销毁、转移等。在Rust里变量与其拥有的资源之间也是ownership，即变量是owner，它对其拥有的资源享有所有权。

Ownership

变量对其拥有的资源享有所有权，比如下面这段代码，涉及多个变量，存在多个ownership。

代码运行后，在内存中，变量之间的关系看起来会像下面这样。

在这张图中，owner和其拥有的资源之间的关系，形成了一个树状结构。

  1. composer 拥有一个vector

  2. vector 拥有多个 Person 元素

  3. Person 拥有 name 和 birth 字段

  4. name 是String，拥有buffer、capacity、length及buffer指向的文本

每棵树的root是个变量，比如上图的root是 composer ，当代码运行超出root的作用域后，以root为根的整棵树将会被销毁。

Rust不像C或C++使用free、delete等操作对变量进行显式销毁，Rust销毁变量的方式是自动的。当代码运行超出变量作用域，或从vector里删除了一个元素，或类似的情况发生时，Rust从ownership tree上删除该变量从而销毁该变量。

Move

在Rust里，对于大多数类型，将值赋给变量、传递给函数或从函数返回时，都是对值进行 move ，而不是 copy 。原来的owner将ownership转交给新的owner，由新的owner控制变量的生命周期。

Rust采用了 Move 语义来处理变量赋值等操作，本质上是采用了一种和其它语言不同的内存管理思路，以采用 引用计数 为例的Python来看如下代码。

在Rust里必须显式给出要执行copy的代价，如果想得到像上述C++类似的效果，可以通过如下代码实现，可以看到这要求开发人员必须明确知道这个操作的代价是copy。

  fn main() {
      let s = vec!["udon".to_string(), "ramen".to_string(), "soba".to_string()];
      let t = s.clone();
      let u = s.clone();
  }

发生move的操作

如下代码，有多个操作都发生了move操作。

1
2
3
4
5

  struct Person { name: String, birth: i32 }

  let mut composers = Vec::new();
  (Person { name: "Palestrina".to_string(),
                          birth: 1525 });

  • 从函数返回值

    • Vec::new() 构造了一个vector并返回了自身，ownership从Vec::new move 给了变量composers

  • 构造新值

    • Person 的 name 字段，通过 "Palestrina".to_string() 进行了初始化，ownership从to_string move 给了Person结构体

  • 将值传递给函数

    • 在执行composers.push时，Person作为参数传递给了push方法，ownership从Person move 给了composer这个vector

Copy(Move的例外)

上面提到的vector、string等类型，如果执行copy语义的话，有可能会使用大量内存，成本较高，因此使用move语义会更高效。对于一些简单的类型比如interger、character等，它们在内存中仅仅是简单的bit模式(不涉及复杂的指针引用)，这些类型没有必要执行move语义，因此这些类型Rust将其作为move语义的例外，可以称为 Copy 类型，实际上它只是实现了 Copy 这个trait，只有简单到可以通过bit-for-bit完成copy的类型可以实现 Copy trait。

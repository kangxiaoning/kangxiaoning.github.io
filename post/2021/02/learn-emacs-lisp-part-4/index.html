REPL

spacemacs 内置了一个 REPL(Read-Eval-Print-Loop) ，可以基于这个环境快速验证一些简单的代码。打开方式 是键入 SPC SPC ，在弹出的窗口输入 ielm 回车。

变量

Lisp 中一个 symbol 既可以关联一个值，也可以关联一个函数定义。如果关联的是一个值，则叫变量，变量值可以 是 symbol, number, list, string 。

在 Lisp 中变量和函数定义属于不同的命令空间，因此一个 symbol 可以同时关联值和函数定义。

打印变量

在 REPL 中输入变量名回车即可打印变量，比如 Emacs 的 fill-column 变量在我这里是80。

1
2
3

  ELISP> fill-column
  80 (#o120, #x50, ?P)
  ELISP>

void variable

如果一个执行一个没有绑定变量的 symbol ，则会报 void symbol 错误。

1
2
3

ELISP> +
*** Eval error ***  Symbol’s value as variable is void: +
 ELISP>

void function

如果一个 symbol 是列表中的第一个元素或者仅有的一个元素， Lisp interpreter 将把当作函数并尝试 去找函数定义，此时如果找不到，则会报 void function 错误。

1
2
3

ELISP> (fill-column)
*** Eval error ***  Symbol’s function definition is void: fill-column
 ELISP>

参数

在 Lisp 中，函数也是 symbol ，因此下面的 + 实际指向了一个函数定义，函数的参数即为 + 后面的数字。

1

  (+ 2 2)

message function

message 函数可以接受多个参数，功能是将参数输出到 echo area 同时也写入到 Message bufffer中。

prefix

Lisp 采用的是 prefix 表达式，因此操作符在参数前面，关于表达式的三种写法可以参考如下，虽然写法不同，但效果 是一样的。

Infix, Postfix and Prefix notations are three different but equivalent ways of writing expressions. It is easiest to demonstrate the differences by looking at examples of operators that take two operands1.

  • Infix notation: X + Y

  • Postfix notation (also known as "Reverse Polish notation"): X Y +

  • Prefix notation (also known as "Polish notation"): + X Y

绑定变量

绑定定量有如下几种方式，下面介绍下 set 和 setq 的用法。

  1. set

  2. setq

  3. let

set

上一篇提到了单引号的作用，如果 symbol 前面不带单引号，则解释器会将其看作一个命令去执行，执行完成后再执行 set 。

这里可以看到 set 也是一个 list ，第一个元素 set 没有带单引号，因此会被解释器执行，后面两个元素带了单引号，所以 会以字面文本来处理，效果就是把第三个元素作为value绑定给第二个元素。

  • 正确写法

1

  (set 'flowers '(rose violet daisy buttercup))

  • 错误写法

    错误原因是变量名前面没有带单引号。

1

  (set flowers '(rose violet daisy buttercup))

setq

由于 set 的使用需要第一个参数以单引号引用，这种场景经常出现，因此诞生了 setq ， q 在这里的意思是 quote ，和 set 区别是会自动给第一个参数加引号，所以我们不需要自己写单引号了。还有一个便利是 setq 允许一次设置多个变量。

1

  (setq carnivores '(lion tiger leopard))
1

  (set 'carnivores '(lion tiger leopard))

上面两个是等价的写法。

下面是一次设置多个变量的示例，可以在一行，也可以在多行， Lisp 在解释执行时会处理掉多余的空白符。

1
2

  (setq trees '(pine fir oak maple)
        herbivores '(gazelle antelope zebra))

let

In computer science jargon, we would say the binding of a symbol is visible only in functions called in the let form; in Emacs Lisp, the default scoping is dynamic, not lexical2.

let 创建变量的作用域在 let 表达式内，退出 let 表达式后变量自动销毁，可以理解为 setq 创建的临时/本地变量。 Emacs Lisp 默认的作用域是动态的， 而不是 lexical 的。~let~ 可以一次创建多个变量并初始化，不显式初始化的变量默认为 nil ， let 的最后一个表达式的值作为 let 表达式的返回值。

一个 let 表达式的模板如下，饱含三个部分。

  1. let 关键字

  2. 变量列表，比如 (thread (needles 3)) ，其中 thread 未显式初始化，值为 nil ； needles 显式初始化为 3

  3. let 表达式 body ，可以包含多个 list

(let ((variable value)
      (variable value)
      …)
  body…)

如下示例对变量进行了初始化，因此 message 打印变量的值为初始化指定的值。

1
2
3
4

  (let ((zebra "stripes")
        (tiger "fierce"))
    (message "One kind of animal has %s and another is %s."
             zebra tiger))

如下示例未对中间2个变量进行初始化，因此 message 打印变量的值为 nil 。

1
2
3
4
5
6
7

  (let ((birch 3)
        pine
        fir
        (oak 'some))
    (message
     "Here are %d variables with %s, %s, and %s value."
     birch pine fir oak))

let*

在某些情况下，要根据一个变量初始化另一个变量，此时let无法满足需求，比如下面这段代码，运行会报错 (void-variable y) 。

1
2
3

  (let ((y 5)
        (z (+ y 5)))
    (* y z))

上面这段代码中，变量z需要根据y来计算，此时使用 let 可以解决问题。

1
2
3

  (let* ((y 5)
         (z (+ y 5)))
    (* y z))

let* 实际上是做了一些展开，如下所示。

1
2
3

  (let ((y 5))
    (let ((z (+ y 5)))
      (* y z)))

setq-local

只在当前buffer设置变量。

1

  (setq-local tab-width 4)

make-variable-buffer-local

You can make any variable local for all future buffers with the make-variable-buffer-local function:

1
2

  (setq not-local-yet t)
  (make-variable-buffer-local 'not-local-yet)

总结

这两篇学习了一些 lisp 基础知识，如果将学习 lisp 比作爬山，第一部分是最陡峭的，我们已经爬过了最困难 的部分，接下来会相对容易些，下面对前面所学做个总结。

  • Lisp programs are made up of expressions, which are lists or single atoms.

  • Lists are made up of zero or more atoms or inner lists, separated by whitespace and surrounded by parentheses. A list can be empty.

  • Atoms are multi-character symbols, like forward-paragraph, single character symbols like +, strings of characters between double quotation marks, or numbers.

  • A number evaluates to itself.

  • A string between double quotes also evaluates to itself.

  • When you evaluate a symbol by itself, its value is returned.

  • When you evaluate a list, the Lisp interpreter looks at the first symbol in the list and then at the function definition bound to that symbol. Then the instructions in the function definition are carried out.

  • A single-quote ‘'’ tells the Lisp interpreter that it should return the following expression as written, and not evaluate it as it would if the quote were not there.

  • Arguments are the information passed to a function. The arguments to a function are computed by evaluating the rest of the elements of the list of which the function is the first element.

  • A function always returns a value when it is evaluated (unless it gets an error); in addition, it may also carry out some action that is a side effect. In many cases, a function’s primary purpose is to create a side effect3.

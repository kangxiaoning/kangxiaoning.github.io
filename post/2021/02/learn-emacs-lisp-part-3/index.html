Editing Types

如下类型是Emacs Lisp提供的，用于编辑目的的附加类型。

TypesComment
BufferThe basic object of editing.
MarkerA position in a buffer.
WindowBuffers are displayed in windows.
FrameWindows subdivide frames.
TerminalA terminal device displays frames.
Window ConfigurationRecording the way a frame is subdivided.
Frame ConfigurationRecording the status of all frames.
ProcessA subprocess of Emacs running on the underlying OS.
ThreadA thread of Emacs Lisp execution.
MutexAn exclusive lock for thread synchronization.
Condition VariableCondition variable for thread synchronization.
StreamReceive or send characters.
KeymapWhat function a keystroke invokes.
OverlayHow an overlay is represented.
FontFonts for displaying text.

能直观感觉到的类型有Frame、Window、Buffer，比如可以对这几种类型执行创建/删除等操作，用户使用比较多的是Window和Buffer，比如分屏就是创建了Window，在Window里编辑文件就是在操作一个Buffer，下图简单描述了三者的关系。

Buffer

buffer是一个持有可编辑文本的对象，大多数buffer会关联到一个磁盘上的文件，也有一些没有关联到文件；大多数buffer会在一个window里展现给用户，但是buffer不是必须在window里展示；很多命令都是对buffer内容进行的操作。任何时候我们正在操作的buffer叫做 current buffer ，比如执行下面代码，会打印出我正在编辑的buffer名称 #<buffer learn-emacs-lisp-part-3.org> ，可以看到这是一个文件名。

1

  (current-buffer)

每个buffer都有几种与之关联的数据结构

  • a local syntax table

  • a local keymap

  • a list of buffer-local variable bindings

  • overlays

  • text properties for the text in the buffer

本地keymap和变量包含的条目会覆盖全局设置，主要用来自定义某个buffer的行为，而不影响其它buffer。

Marker

marker表示buffer中的位置。比如执行下面代码会打印出 #<marker at 3031 in learn-emacs-lisp-part-3.org> 。

1

  (point-marker)

Window

如上面图中所示，window是屏幕的一个部分，用来显示buffer的内容，每个window都关联一个buffer。反过来说，一个buffer可以出现在一个window或多个window，也可以不出现在window里。

每个window属于并且只属于一个frame，光标所在的window叫做 selected window ，比如执行下面代码会打印出 #<window 3 on learn-emacs-lisp-part-3.org> 。

1

  (selected-window)

Frame

frame是屏幕的一个区域，包含一个或多个window。我经常是全屏使用emacs，因此一般这个frame就是屏幕的全部区域，执行如下代码可以打印出当前所在的frame #<frame Emacs-x86_64-10_14@kangxiaingdeMBP 0x7fe92a00fc70> 。

1

  (selected-frame)

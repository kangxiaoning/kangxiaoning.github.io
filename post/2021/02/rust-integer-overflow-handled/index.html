Rust integer overflow handled

Rust 中 u{bit_width}/i{bit_width} 等类型是通过固定宽度的 bit 来表示数字的，当算术操作的结果超出类型的 bit width 能表达的数字范围时会发生 overflow ，在debug build中Rust会检查到这种情况并报错。

有时这种对 overflow 的默认处理规则不能满足我们的需求，因此 Rust 也提供了其它方法供我们选择，大概可以归为三类。

overflow

如下模拟一个 overflow 的情况，在执行 cargo run 后 Rust 报了 overflow 的编译错误。

下面分别介绍 Rust 给出的三种 overflow 处理方法。

handled

Checked

checked 操作返回 Option 类型，当结果在类型可表示范围内时返回 Some(v) ，当结果不在类型可表示范围内时返回 None 。

fn main() {
    // The sum of 10 and 20 can be represented as a u8.
    assert_eq!(10_u8.checked_add(20), Some(30));

    // Unfortunately, the sum of 100 and 200 cannot.
    assert_eq!(100_u8.checked_add(200), None);

    // Do the addition; panic if it overflows.
    // let sum = x.checked_add(y).unwrap();

    // Oddly, signed division can overflow too, in one particular case.
    // A signed n-bit type can represent -2ⁿ⁻¹, but not 2ⁿ⁻¹.
    assert_eq!((-128_i8).checked_div(-1), None);
}

Wrapping

wrapping 操作返回的结果，等价于 数学上正确的结果 对该 类型值域总数 执行 取模 的结果。

// The first product can be represented as a u16;
// the second cannot, so we get 250000 modulo 2¹⁶.
assert_eq!(100_u16.wrapping_mul(200), 20000);
assert_eq!(500_u16.wrapping_mul(500), 53392);

// Operations on signed types may wrap to negative values.
assert_eq!(500_i16.wrapping_mul(500), -12144);

// In bitwise shift operations, the shift distance is wrapped to fall within
// the size of the value. So a shift of 17 bits in a 16-bit type is a shift
// of 1.
assert_eq!(5_i16.wrapping_shl(17), 10);

以 500_i16.wrapping_mul(500) 的计算过程为例进行解释。

  1. 500 x 500 = 250000，得到 数学上正确的结果

  2. i16 类型有 2^16 个值，即65536，得到 类型值域总数

  3. 执行 取模 操作，250000 % 65536 = 53392

  4. 此时该数字一定可以用 16 bits 二进制表示，将二进制解释为 i16 类型即得到最终结果 -12144

    • 53392 的二进制为 1101000010010000

    • 1101000010010000 表示 i16 对应的十进制是 -12144

Saturating

saturating 操作返回该类型可示范围内，最接近数学上正确结果的值。

1
2

assert_eq!(32760_i16.saturating_add(10), 32767);
assert_eq!((-32760_i16).saturating_sub(10), -32768);

上述加法和减法，在数学上的正确结果超出了i16类型可表示的范围，因此结果是i16类型的最大值和最小值。

handled operation names

三种操作对应的方法名称以 checked_, wrapping_, saturating_ 前缀开始，接下面第二列的后缀组成对应操作的方法名。

OperationName SuffixExample

Subtractionsub10_u8.checked_sub(11) == None

Remainderrem(-32768_i16).wrapping_rem(-1) == 0
Negationneg(-128_i8).checked_neg() == None
Absolute valueabs(-32768_i16).wrapping_abs() == -32768
Exponentiationpow3_u8.checked_pow(4) == Some(81)

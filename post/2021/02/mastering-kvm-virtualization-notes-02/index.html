虚拟化是一种将硬件与软件解耦的方法，它提供了一种高效的、可编程的切分资源和共享资源的方法，本篇了解libvirt、QEMU、KVM原理及通信。

不能虚拟化的原因

不能虚拟化可能有多种原因，最简单的是 虚拟化技术不支持应用运行所需的OS 。

虚拟化硬件要求

  • Second-Level Address Translation, Rapid Virtualization Indexing, Extended Page Tables (SLAT/RVI/EPT) support

    • 这是CPU相关的技术，hypervisor可以使用这种技术进行虚拟内存地址到物理内存地址的转换，可以降低虚拟机访问内存的延迟。SLAT在Inter的实现叫EPT，在ADM的实现叫RVI，它们是通过 Translation Lookaside Buffer(TLB) 实现的。

  • Intel VT or AMD-V support

    • 具有VT的Inter CPU或具有AMD-V的ADM CPU，表示它们支持硬件虚拟化扩展(hardware virtualization extensions)及全虚拟化(full virtualization)。

  • Long mode support

    • 指64-bit CPU。主要原因是32-bit CPU最多只能管理4G内存，没有实际价值。

  • The possibility of having Input/Output Memory Management Unit (IOMMU) virtualization (such as AMD-Vi, Intel VT-d, and stage 2 tables on ARM)

    • 允许虚拟机直接访问外围的硬件，比如显卡、存储控制器、网络设备等。

  • The possibility to do Single Root Input Output Virtualization (SR/IOV)

    • 利用这个能力可以直接让PCIE设备转发到多个虚拟机。SR-IOV的关键是通过 Virtual Functions (VFs) 让多个虚拟机共享一个物理设备。

  • The possibility to do PCI passthrough

    • 服务器上的PCIE卡利用 Physical Functions (PFs) 技术，可以直接让虚拟机使用，就像虚拟机直接连接该卡一样。

  • Trusted Platform Module (TPM) support

    • 通常以额外的芯片实现，使用TMP可以提供安全性，因为其可以提供加密支持。

虚拟化软件要求

计算机科学领域存在各种分层的保护域或特权环，用来保护计算机资源以提升系统安全性。如果将这些保护环想像成指令区域，可以用下图表示。

  • ring 0是最高权限级别，可以直接访问硬件资源，比如CPU、内存等

  • ring 1和ring 2基本没有使用

  • ring 3运行应用级别的代码，如果要执行特权操作，必须通过system call来执行

  • Linux只使用了2个ring，kernel运行在ring 0，user mode运行在ring 3

VMM作为虚拟化管理层，需要访问CPU、内存、I/O设备这些硬件资源，所以要和宿主机kernel一起运行在ring 0；虚拟机上的kernel和宿主机的kernel一样，也会认为自己运行在ring 0；然而只能有一个kernel运行在ring 0，这种情况下虚拟化就面临一个如何处理这种冲突的难题，也因此出现了三种不同的技术实现。

FULL VIRTUALIZATION

在Full Virtualization方案中，Guest OS运行在ring 1，VMM运行在ring 0。因为Guest OS运行在ring 1，没有权限执行特权指令，因此在执行特权指令时会触发异常，在这个时机由VMM捕获异常并执行二进制翻译，再模拟特权指令并执行后返回给Guest OS。

  • 优点是Guest OS无需做任何个性

  • 缺点是性能损耗大

PARAVIRTUALIZATION

前面提到这个问题主要是如何处理VMM和Guest OS运行在ring 0上，在Full Virtualization方案中选择了让Guest OS运行在ring 1，然后VMM来翻译特殊指令。那是不是也可以修改Guest OS替换掉有冲突的特权指令呢？没错，Paravirtualization方案正是按这个思路来实现的。

在Paravirtualization方案中，需要修改Guest OS，通过hypercall完成和VMM的通信。

  • 优点是性能损耗小，因为省去了Full Virtualization方案中的异常捕获、指令翻译等过程

  • 缺点是需要修改Guest OS，Xen不支持Windows的原因就是因为无法修改Windows代码

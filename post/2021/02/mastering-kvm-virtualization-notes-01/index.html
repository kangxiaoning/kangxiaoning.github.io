Matering KVM Virtualization - Second Edition读书笔记。

虚拟化技术并不是x86或Linux的新技术，早在1967年IBM CP-40和S/360-40等非x86产品已经存在虚拟化技术。

开源市场的最大玩家，RedHat于2007年在 RHEL 5 中引入 Xen 虚拟化，之后由于某些原因放弃了 Xen ，于2010年在 RHEL 6 中转向了 KVM ，在此之后主要基于 KVM 发展虚拟化技术。

虚拟化类型

根据不同维度可以有不同的分类，如下简单做个分类。

虚拟化内容

如果是基于虚拟化的对象，有如下分类。

  1. Desktop virtualization (Virtual Desktop Infrastructuree (VDI))

  2. Server virtualization

  3. Application virtualization

  4. Network virtualization (and a more broader, cloud-based concept called Software-Defined Networking (SDN))

  5. Storage virtualization (and a newer concept Software-Defined Storage (SDS))

如何虚拟化VM

如果是谈论如何虚拟化一个虚拟化，则有下面的分类。

  1. Partitioning

  2. Full virtualization

    • Software-based

    • Hardware-based

  3. Paravirtualization

  4. Hybrid virtualization

  5. Container-based virtualization

Type1和Type2

如果VMM/Hypervisor直接运行上硬件之上被归类type1 hypervisor；如果VMM/Hypervisor运行于OS之上被归类为type2。这种划分没有明确的、标准的定义。

Programming Rust 2nd Edition 读书笔记。

Rust 是类型安全（type-safe）的语言，那么类型安全具体是什么意思？

Type Safety

undefined behavior

首先得说一下 undefined behavior ， C99 标准对 undefined behavior 的定义如下。

behavior, upon use of a nonportable or erroneous program construct or of erroneous data, for which this International Standard imposes no requirements

因此，如下数组越界的行为在 C 语言是合法的，但是运行结果不可预知，比如下面代码在不同机器上的运行结果可能完全不一样，被视为未定义行为。

int main(int argc, char **argv) {
  unsigned long a[1];
  a[3] = 0x7ffff7b36cebUL;
  return 0;
}

type safe

根据上面的 undefined behavior 的理解， type safe 定义如下。

If a program has been written so that no possible execution can exhibit undefined behavior, we say that program is well defined. If a language’s safety checks ensure that every program is well defined, we say that language is type safe.

type safe 强调语言自身对于 undefined behavior 的检查能力，如果语言自身的检查能确保不出现 undefined behavior ，那么则说这个语言是类型安全的。虽然 C 也能写出 well defined 的程序，但是语言自身不能确保不出现 undefined behavior ，所以 C 不是类型安全的语言。

相比之下 Python 则是类型安全的，因为 Python 语言自身具备检查能力确保不会出现类似数组越界的 undefined behavior 。

注意类型安全与语言在编译期检查还是运行期检查无关， C 语言属于编译时检查，却并非类型安全； Python 属于运行时检查，却是类型安全的。

C 和 C++ 作为系统编程领域的主流语言，不能确保类型安全，然后大多数其它语言却是类型安全的，是有一些讽刺的。而这也是 Rust 致力于解决的问题，兼备类型安全和系统编程语言的优点。

与 Python ， JavaScript 等动态语言相比， Rust 需要开发人员在写代码前做更多的规划，比如先弄清楚函数参数的类型、返回值类型、 struct 字段类型等，看起来很麻烦但是 Rust 提供了两个特性让这些事情变得比预期简单。

类型推断

类型推断只需要我们定义一次类型，代码其它地方编译器会帮我们完成推断，避免写大量类型代码，提升了代码的可读性同时又保持了编译期间检查类型错误的能力。

fn build_vector() -> Vec<i16> {

    v.push(10);
    v.push(20);
    v
}

泛型

Rust 支持泛型，一个函数可以处理多种不同的类型。 Python 等动态语言的 duck typing 也具备这种灵活性，但这些动态语言难以在运行之前发现类型错误，通常需要编写大量测试代码来捕获类型错误。而 Rust 则不同，既具备泛型的灵活性同时又可以在编译期间发现类型错误。

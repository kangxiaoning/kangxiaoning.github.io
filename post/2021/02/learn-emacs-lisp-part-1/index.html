Emacs Lisp

从其名称来看首先是个 Lisp ，其次和 Emacs 有关。没错，它就是用来配置和扩展 Emacs 的 Lisp 编程语言，Emacs很多功能都是用它编写的，因此要熟练使用和扩展Emacs，学习Emacs Lisp是必须的。

既然是Lisp，我们先了解下Lisp。

Lisp

参考 wiki 解释，Lisp是个编程语言家族，具有全括号前缀表示法的特点，诞生于1958年，是迄今为止仍在广泛使用的第二大古老的高级编程语言。Fortran是最古老的，比Lisp早一年。当前比较著名的通用Lisp语言有Racket、Common Lisp、Scheme、Clojure等。

Lisp stands for LISt Processing, and the programming language handles lists (and lists of lists) by putting them between parentheses. Lists are the basis of Lisp1.

如上引用所示，Lisp这个名字来自List Processing的缩写，顾名思义，它是处理列表的一门语言。Linked list是这门语言的主要数据结构之一，并且Lisp语言的源代码也是由list组成的，因此Lisp程序可以将Lisp语言作为数据进行操作，从而产生了它强大的宏系统，允许程序员在Lisp里创建新的语法或者DSL(domain-specific languages)。

Lisp的语法非常简单，由 S-expressions 组成。S-expressions的全称是symbolic expression，简写为sexprs，由atom或list组成，所以接下来我们了解下list和Atom。

The single apostrophe, ', that I put in front of some of the example lists in preceding sections is called a quote; when it precedes a list, it tells Lisp to do nothing with the list, other than take it as it is written. But if there is no quote preceding a list, the first item of the list is special: it is a command for the computer to obey2.

上面我们也了解到Lisp是处理list的，而Lisp本身又是由list组成的，所以在Lisp里数据和程序的表现方式相同，也就是说他们都是words 、numbers等元素组成的list ，或是子list组成的list，这些list元素由空白符分隔，括号包围。

也正是这个特性，使一个程序可以很容易作为另一个程序的数据，是Lisp中非常强大的一个特性。

Atoms

Atoms，如其名字所表达的意思一样，意为不可分割。从技术角度来说，Lisp里的list是由括号、空白符、Atoms、或者其它括号、空白符、Atoms组成的子list组成，所以list里的元素就是Atoms或子list。一个不包含任何元素的list 记作 () ，称为 empty list 。

比如下面几种类型都是Atoms。

  • numbers ，如 33，44，55

  • symbols ，如 '+', 'foo', 'forward-line'

  • string ，由双引号包围的文本，如 '(this list includes "text between quotation marks.")

运行程序

Lisp是解释执行的语言，因此需要相应的解释器来运行Lisp代码，而Emacs除了是个编辑器外，还自带了一个由C语言编写的Lisp解释器，所以接下来我们在Emacs里运行Emacs Lisp代码，来了解解释器的行为。

Lisp Interpreter

从前面我们知道Lisp就是处理list的，其语法是s-expression形式，而s-expression又是由Atoms或list组成的，因此Lisp中的list就是随时可以运行（在Lisp中称为 evaluate ）的程序，运行后会有如下3种可能的情况发生。

  1. 返回自身（以单引号开头的list）

  2. 把第一个 symbol 当作命令，其它作为输入进行处理

  3. 抛出错误

通常第2种是我们想要的结果。

Interpreter可以解释源码，也可以解释字节码，源码文件以 .el 后缀结尾，字节码文件以 .elc 后缀结尾。

evaluate a list

在Emacs里，运行一个list，将光标行动到list最右侧括号，按下 C-x C-e 即可在屏幕底部看到结果，有十进制、八进制、十六进制、字符等多种展示。

Evaluating Inner Lists

List可以嵌套使用，最内层的list最早执行，返回结果作为外层list的输入。比如下面的表达式， (+ 3 3) 先执行，然后返回6，再执行 (+ 2 6) 最终结果为8。

1

  (+ 2 (+ 3 3))

运行上述代码，本质上是将这些代码传递给 Lisp interpreter 来执行的， Lisp interpreter 将人可以理解的语言转换为机器可以理解的语言，执行后再将结果以能理解的形式展示出来。

当我们按下 C-x C-e 时，实际是对 eval-last-sexp 这个命令做了一次交互式调用， eval 是 evaluate 的缩写， sexp 是 symbolic expression 的缩写， 这个命令的意思是执行最后一个符号表达式，即光标前面的表达式。

如上错误的第一行已经告诉我们原因， void-function this ，因为这个 List 没有以单引号开头，解释器把第一个 atom 当作一个命令去调用，然后这个命令是没有定义过的，因此被认为是 void 的。

使用 spacemacs 快两年时间了，用 Org 写文档，用 org-re-reveal 写PPT，用 lsp 写代码，越来越喜欢这种不用鼠标就可流畅操作的体验。 作为一款以扩展性著称的编辑器，必须掌握 emacs lisp 才能发挥它的威力，加之自己打算长期使用，因此计划花时间学习一下。

Lisp stands for LISt Processing, and the programming language handles lists (and lists of lists) by putting them between parentheses. Lists are the basis of Lisp1.

基础概念

Emacs Lisp 是什么？

Emacs Lisp 是用来配置和扩展 Emacs 的 lisp 编程语言，Emacs很多功能都是用它编写的。

Lisp 语法

Lisp语法非常简单，主要由 lists, symbol, value 组成，括号多是它的一大特点。

Emacs Types

Emacs Lisp中任何 value 或者 object 都有一个类型，也有其文本表示形式，有的是可以阅读的，有的是不可阅读的。Emacs Lisp的类型有两大类，一类是用于Lisp编程，一类是Emacs编辑相关的，后者是Emacs Lisp特有的类型。

Cons Cell

  • cons cell 由两个 slots 组成，每个 slot 都可以保存任何Lisp对象，分别称为 CAR 和 CDR

  • List就是由多个 cons cell 连接起来构建出来的，可以理解为 linked list

  • 通过 car 和 cdr 访问 cons cell

    • car 访问 cons cell 的第一个元素

    • cdr 访问 cons cell 的第二个元素

The names CAR and CDR derive from the history of Lisp. The original Lisp implementation ran on an IBM 704 computer which divided words into two parts, the address and the decrement; CAR was an instruction to extract the contents of the address part of a register, and CDR an instruction to extract the contents of the decrement. By contrast, cons cells are named for the function cons that creates them, which in turn was named for its purpose, the construction of cells.

如下是一个由3个元素组成的list (rose violet buttercup) 的图示，写法等价于 (rose . (violet . (buttercup))) 。

   +--- ---+    +--- ---+    +--- ---+
   |   |   |--> |   |   |--> |   |   |--> nil
   +--- ---+    +--- ---+    +--- ---+
     |            |            |
     |            |            |
      --> rose     --> violet   --> buttercup

下图是相同list的另一种图示。

+---------------+     +----------------+     +-------------------+
| car   | cdr   |     | car    | cdr   |     | car       | cdr   |
| rose  |   o-------->| violet |   o-------->| buttercup |  nil  |
|       |       |     |        |       |     |           |       |

List

在 Lisp 中，数据和程序的表现方式相同，也就是说他们都是 words 、 numbers 组成的 List ，或 List 组成的 List ，由空白符分隔，括号包围。 也正是这个特性，一个程序可以很容易作为另一个程序的数据，是 Lisp 中非常强大的一个特性。

The single apostrophe, ', that I put in front of some of the example lists in preceding sections is called a quote; when it precedes a list, it tells Lisp to do nothing with the list, other than take it as it is written. But if there is no quote preceding a list, the first item of the list is special: it is a command for the computer to obey2.

Whitespace in Lists

List 中空白符的数量不重要，多余的空白符主要用来提升代码可读性， Lisp 在解释这些表达式时会删除多余的空白符3。

如下两个 List 在 Lisp 是相同的。

1

  '(this list
         looks like this)

  '(this list looks like this)

In brief, a list is between parentheses, a string is between quotation marks, a symbol looks like a word, and a number looks like a number.3

Atoms

在 Lisp 中， words 称为 Atoms ，意为不可分割。一个不包含任何元素的 List 记作 () ，称为 empty list 。

List 中有如下几种 Atoms 。

  • numbers ，如 33，44，55

  • symbols ，如 '+', 'foo', 'forward-line'

  • string ，由双引号包围的文本，如 '(this list includes "text between quotation marks.")

运行程序

Lisp Interpreter

Lisp 中的 List 表示随时可以运行（在Lisp中称为 evaluate ）的程序，运行后会有如下3种可能的情况发生。

  1. 返回自身（以单引号开头的list）

  2. 把第一个 symbol 当作命令，其它作为输入进行处理

  3. 抛出错误

通常第2种是我们想要的结果。

Interpreter可以解释源码，也可以解释字节码，源码文件以 .el 后缀结尾，字节码文件以 .elc 后缀结尾。

evaluate a list

在 Emacs 里，运行一个 List ，将光标行动到最右侧括号，按下 C-x C-e 即可在屏幕底部看到结果，有十进制、八进制、十六进制、字符等多种展示。

2
3

  (+ 8 9)
  (* 9 9)
  '(你好 Emacs)

Evaluating Inner Lists

List 可以嵌套使用，最内层的 List 最早执行，返回结果作为外层 List 的输入。比如下面的表达式， (+ 3 3) 先执行，然后返回6，再执行 (+ 2 6) 最终结果为8。

1

  (+ 2 (+ 3 3))

运行上述代码，本质上是将这些代码传递给 Lisp interpreter 来执行的， Lisp interpreter 将人可以理解的语言转换为机器可以理解的语言，执行后再将结果以能理解的形式展示出来。

当我们按下 C-x C-e 时，实际是对 eval-last-sexp 这个命令做了一次交互式调用， eval 是 evaluate 的缩写， sexp 是 symbolic expression 的缩写， 这个命令的意思是执行最后一个符号表达式，即光标前面的表达式。

如上错误的第一行已经告诉我们原因， void-function this ，因为这个 List 没有以单引号开头，解释器把第一个 atom 当作一个命令去调用，然后这个命令是没有定义过的，因此被认为是 void 的。

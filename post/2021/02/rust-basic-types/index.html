Rust 是类型安全（type-safe）的语言，那么类型安全具体是什么意思？ Rust 基本类型有哪些？本篇做个大概的介绍。

Type Safety

undefined behavior

首先得说一下 undefined behavior ， C99 标准对 undefined behavior 的定义如下。

behavior, upon use of a nonportable or erroneous program construct or of erroneous data, for which this International Standard imposes no requirements

因此，如下数组越界的行为在 C 语言是合法的，但是运行结果不可预知，比如下面代码在不同机器上的运行结果可能完全不一样，被视为未定义行为。

1
2
3
4
5

int main(int argc, char **argv) {
  unsigned long a[1];
  a[3] = 0x7ffff7b36cebUL;
  return 0;
}

type safe

根据上面的 undefined behavior 的理解， type safe 定义如下。

If a program has been written so that no possible execution can exhibit undefined behavior, we say that program is well defined. If a language’s safety checks ensure that every program is well defined, we say that language is type safe.

type safe 强调语言自身对于 undefined behavior 的检查能力，如果语言自身的检查能确保不出现 undefined behavior ，那么则说这个语言是类型安全的。虽然 C 也能写出 well defined 的程序，但是语言自身不能确保不出现 undefined behavior ，所以 C 不是类型安全的语言。

相比之下 Python 则是类型安全的，因为 Python 语言自身具备检查能力确保不会出现类似数组越界的 undefined behavior 。

注意类型安全与语言在编译期检查还是运行期检查无关， C 语言属于编译时检查，却并非类型安全； Python 属于运行时检查，却是类型安全的。

C 和 C++ 作为系统编程领域的主流语言，不能确保类型安全，然后大多数其它语言却是类型安全的，是有一些讽刺的。而这也是 Rust 致力于解决的问题，兼备类型安全和系统编程语言的优点。

与 Python ， JavaScript 等动态语言相比， Rust 需要开发人员在写代码前做更多的规划，比如先弄清楚函数参数的类型、返回值类型、 struct 字段类型等，看起来很麻烦但是 Rust 提供了两个特性让这些事情变得比预期简单。

类型推断

类型推断只需要我们定义一次类型，代码其它地方编译器会帮我们完成推断，避免写大量类型代码，提升了代码的可读性同时又保持了编译期间检查类型错误的能力。

泛型

Rust 支持泛型，一个函数可以处理多种不同的类型。 Python 等动态语言的 duck typing 也具备这种灵活性，但这些动态语言难以在运行之前发现类型错误，通常需要编写大量测试代码来捕获类型错误。而 Rust 则不同，既具备泛型的灵活性同时又可以在编译期间发现类型错误。

Machine Types

Rust 类型系统的基础是固定宽度数字类型的集合，数字类型遵循常规模式，以 bit 为单位表示其范围及书写形式。

Size (bits)Unsigned integerSigned integerFloating-point
8u8i8
16u16i16
32u32i32f32
64u64i64f64
128u128i128
Machine wordusizeisize

注：这里 的 Machine word 是指运行代码的机器地址位长，比如 32 或 64 位。

Integer Types

  • Rust 使用 u8 表示 byte 值，比如从二进制文件或者 socket 读取的是 u8 流。

  • Rust 里 char 和数字类型区别对待， char 既不是 u8 也不是 i8 ，虽然如此但 Rust 也为 u8 类型提供了类似字符的表示形式， 比如 b'A' 表示 65_u8 ，仅 ASCII 字符可以用这种表示形式。有些字符仅通过单引号来表示可能会引起语义模糊或难以阅读， 如下字符只能通过反斜杠转义来表示。

    CharacterByte literalNumeric equivalent
    Single quote, 'b'\''39u8
    Backslash, \b'\\'92u8
    Newlineb'\n'10u8
    Carriage returnb'\r'13u8
    Tabb'\t'9u8

  • 整数可以通过后缀表示其类型，比如 42u8 表示 u8 ， 1729isize 表示 isize 。

  • 0x , 0o , 0b 前缀分别表示十六进制、八进制、二进制。

  • 可以在数字间增加下线线提升可读性，比如 4_294_967_295 ， 0xffff_ffff ， 127_u8 。

Floating-Point Types

Rust 提供了IEEE标准的单、双精度浮点数。

  • 标准库文档的 f32 (primitive type)” 和 f64 (primitive type) 部分查询相关方法。

bool Type

bool 类型，有两个值， true 和 false ，内存占用一个byte，因此可以创建point指向bool类型。

Characters

char 类型，表示一个 Unicode 字符，内存占用 32-bit。

  • 使用单引号包围字符，例如 ~'中'~。

  • 如果字符在ASCII范围，可以使用字符 Unicode code point 的十六进制来表示，比如 '*' 和 '\x2A'是等价的。

  • 可以将任何 Unicode 字符写为 '\u{HHHHHH}' 。

  • 可以使用 as 将 char 转换为整数类型，如果目标类型小于 32-bits，高位将被truncated

1
2
3

assert_eq!('*' as i32, 42);
assert_eq!('ಠ' as u16, 0xca0);
assert_eq!('ಠ' as i8, -0x60); // U+0CA0 truncated to eight bits, signed

Tuple

tuple 类型，多个值组成的列表，类型可以不同，例如 ("Brazil", 1985) 。

  • zero-tuple () ，又叫 unit type ，常用来表示没有返回值。

  • 函数没有返回值相当于返回值是 () 。

1

fn swap<T>(x: &mut T, y: &mut T);

等价于

1

fn swap<T>(x: &mut T, y: &mut T) -> ();

Point Types

Rust有几种类型用来表示内存地址，这里我们先介绍三种。

Rust被设计为最小化在 heap 上分配内存，Values nest by default，比如 ((0, 0), (1440, 900)) 会被存储为4个相邻的整数，如果它用作本地变量，将会在 stack 上分配4个整数大小的空间，而不需要在 heap 上分配。这么做在内存利用上是很高效的，但也因为这个原因，要在Rust里使用指针必须显式使用指针类型。

Reference

  • &T, A shared reference. 可以一次使用多个shared reference，但是它们是只读的，禁止修改指针指向的值。

  • &mut T, A mutable reference. 可以读写指针指向的值，但是在该指针生命周期内不能存在其它任何指向该值的指针。

Boxes

把一个值分配在堆上最简单的方法就是 Box::new 。

1
2

let t = (12, "eggs");
let b = Box::new(t);  // allocate a tuple in the heap

Raw Pointers

Rust也有raw pointer, *mut T 和 *const T, 使用上和 C++ 相似。使用raw pointer在Rust里是不安全的，因为没有做任何检查。所以只能在 unsafe 语句块里使用raw pointer, unfafe 是Rust的一个高级特性， unsafe 语句块里的代码由开发者来确保其安全性。

trait Foo<const N: usize> {
    fn method<const M: usize>(&mut self, arr: [[u8; M]; N]);
}

struct Bar<T, const N: usize> {
    inner: [T; N],
}

impl<const N: usize> Foo<N> for Bar<u8, N> {
    fn method<const M: usize>(&mut self, arr: [[u8; M]; N]) {
        for (elem, s) in self.inner.iter_mut().zip(arr.iter()) {
            for &x in s {
                *elem &= x;
            }
        }
    }
}

fn function<const N: u16>() -> u16 {
    // Const parameters can be used freely inside of functions.
    (N + 1) / 2 * N
}

fn main() {
    let mut bar = Bar { inner: [0xff; 3] };
    // This infers the value of `M` from the type of the function argument.
    bar.method([[0b11_00, 0b01_00], [0b00_11, 0b00_01], [0b11_00, 0b00_11]]);
    assert_eq!(bar.inner, [0b01_00, 0b00_01, 0b00_00]);

    // You can also explicitly specify the value of `N`.
    assert_eq!(function::<17>(), 153);
}

Vectors

Vec<T>

  • 在 heap 分配内存，动态扩缩容

  • vec! 是宏，等价于 Vec::new 创建个空的vector再push元素

  • 在可预知数组大小的情况下使用 Vec::with_capacity ，避免扩容带来的开销

pal.push("no");
pal.push("pets");
assert_eq!(pal, vec!["step", "on", "no", "pets"]);

Slice

&[T] 和 &mut [T]

  • 因为slice可以表示任何大小，所以不能直接存储在变量里或传递给函数，必须始终通过 reference 传递

  • 是个two-word的fat pointer，一个pointer一个大小，和golang的 string literal 结构一样，golang的slice是3-word的结构

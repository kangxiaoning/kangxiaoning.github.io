Rust Divergent Function

2021-03-07
rust

Contents

一个函数或表达式可能会有多个分支，Rust会检查所有分支的返回类型，确保它们是一致的。比如在 if else 或 match 等表达式里如果两个不同分支的返回类型不一样会得到Rust编译器的错误提示。那像下面的 continue 分支返回的是什么类型？

1
2
3
4

        let guess: u32 = match guess.trim().parse() {
            Ok(num) => num,
            Err(_) => continue,
        };

Never Type

Rust有一个特殊的类型名为 ! ，在类型理论里叫作 empty type 或 zero type ，在Rust里叫 Never Type ，因为它出现的位置正好是函数返回值的位置，但是 ! 表示的是永远不返回。

和Rust里的 unit type () 并不一样， () 是有一个值的，当函数不需要返回有意义的值时，可以返回 () 。

Divergent Function

返回值为 ! 的函数称为 Divergent Function ，表示该函数永远不返回，比如下面的std::process::exit函数。

1
2
3
4

  pub fn exit(code: i32) -> ! {
      crate::sys_common::cleanup();
      crate::sys::os::exit(code)
  }

类型判断

回到文章最开始的问题， continue 返回的是什么类型呢？

正如大家想的那样， continue 返回的是 ! ，其类型是 never type ，当Rust计算上述 guess 的类型时，它会检查两个分支，前一个分支为 u32 ，第二个分支为 ! ，因为 ! 代表没有类型，所以Rust最终确定 guess 的类型是 u32 。

The formal way of describing this behavior is that expressions of type ! can be coerced into any other type

Never type 也同样应用于 panic! 之上，比如下面 unwrap 可以从 Option<T> 中返回一个值或者panic。

  impl<T> Option<T> {
      pub fn unwrap(self) -> T {
          match self {
              Some(val) => val,
              None => panic!("called `Option::unwrap()` on a `None` value"),
          }
      }
  }

类似的用法也出现在 break return loop 等表达式。

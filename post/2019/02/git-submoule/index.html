1.1 背景

在 git 项目中引入第三方库，通过 git clone 下载到项目里，于是产生了 git 嵌套，如果我们要解决如下问题，那么需要考虑 git submodule。

  1. 如何获取第三方库更新？
  2. 如何在 clone 主项目的时候导入第三方库的代码？
  3. 修改了第三方库代码又想继续跟踪第三库的更新怎么办？

2. git submodule 介绍

参考Git Tools Submodules

如下对常用命令做一个记录。

2.1 add submodule

1
2

  cd /path/to/main/project
  git submodule add https://github.com/chaconinc/DbConnector /path/to/submodule

2.2 clone main project

普通 clone 不会将 submodule 的代码下载到本地，可以使用如下方法。

1
2

  # 不加--recurse-submodules 会导致 clone 到本地的 submodule 目录是空目录
  git clone --recurse-submodules https://github.com/chaconinc/MainProject

2.3 delete submodule

删除 submodule 可以在 cached 中删除，如果要变成主项目代码，再将 submodule 目录下的.git 删除即可。 如果要整个删除 submodule 采用如下方法，在主项目 git 里和文件系统都删除。

1
2
3
4

  git rm --cached /path/to/submodule
  rm -rf /path/to/submodule
  # 如果有多个 submodule 需要手动在.gitmodules 文件删除相应条目
  rm .gitmodules

3. 使用案例

blog 项目使用了 hexo 和 next，我需要在多台电脑使用，因此将 blog 项目 push 到仓库，遇到如下问题。

3.1 跟踪 next 更新

要跟踪更新，需要使用 git fetch 来获取更新。

3.2 修改 next 配置

修改了 next 配置，需要保存到远程仓库，在不同电脑使用。

3.3 解决方法

第一个问题，clone next 主题后，保留.git 即可，但是要 push 到远程仓库，涉及到 git submodule 使用。 第二个问题，我修改了_config.yml，需要将变化保存下来，push 到远程仓库，在另一台电脑 clone 使用，这里将 submodule 项目的 push url 修改即可。

1
2

  cd /path/to/theme/next
  git remote set-url --push origin git@github.com:user/theme-next.git

这样，既可以将修改保存下来，也可以在另一台电脑 clone 到修改后的代码，需要更新 next 的时候执行下 git fetch 即可，解决了问题。

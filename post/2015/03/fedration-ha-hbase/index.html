(五) Federation + HA + Hbase搭建

步骤一、 资源规划

Federation架构实现如下。

在已有环境增加一套NameNode组成Federation，规划在datanode3和datanode4上，搭建完成后最终进程分布如下表。

主机名IP用户Hadoop Cluster

步骤二、 清理环境模拟全新搭建

为了降低复杂度，不考虑从已有环境改造，而是全新搭建，理解原理后再考虑升级改造之类的方案。

1) 停止集群

执行如下命令停止集群，检查进程全部停止后进入下一步，zookeeper可以不停止。

1
2
3

stop-hbase.sh
stop-yarn.sh
stop-dfs.sh

2) 清空数据

所有节点执行如下命令，清空数据，忽略目录不存在的报错。

1
2
3

cd /home/hadoop/hadoopdata/hdfs/namenode;ls -l; rm -rf current;ls -l
cd /home/hadoop/hadoopdata/hdfs/datanode;ls -l; rm -rf current;ls -l
cd /home/hadoop/journal/node/local/data;ls -l; rm -rf current;ls -l

步骤三、 配置信任关系

datanode3和datanode4规划为一套NameNode，需要添加信任关系，参照配置SSH信任处理。

**提示：**不需要再生成公钥，只需要将相关的公钥添加到对应主机即可。

步骤五、 格式化NameNode

1) 启动zookeeper

参照启动zookeeper。

2) cluster01格式化zookeeper

格式化操作的目的是在ZK集群中建立一个znode，用于保存集群cluster01中NameNode的状态数据

参照格式化zookeeper,在namenode1执行。

3) cluster02格式化zookeeper

参照格式化zookeeper,在datanode3执行。

4) 启动JournalNode

在namenode1、namenode2、datanode1执行如下命令。

1

hadoop-daemon.sh start journalnode

5) cluster01格式化NameNode

**注意：**如下命令中的clusterID必须是同一个，表示共享DataNode的存储资源，否则不会形成Federation，而是两个独立的HDFS。

在namenode1执行如下命令，格式化并启动NameNode。

1
2

hdfs namenode -format -clusterId hdfscluster01
hadoop-daemon.sh start namenode

在namenode2执行如下命令，完成格式化同步并启动NameNode。

1
2

hdfs namenode -bootstrapStandby
hadoop-daemon.sh start namenode

6) cluster02格式化NameNode

在datanode3执行如下命令，格式化并启动NameNode。

1
2

hdfs namenode -format -clusterId hdfscluster01
hadoop-daemon.sh start namenode

在datanode4执行如下命令，完成格式化同步并启动NameNode。

1
2

hdfs namenode -bootstrapStandby
hadoop-daemon.sh start namenode

7) 初步验证Federation

在浏览器验证4台NameNode节点，下图红框所示的值应该是一样的，否则可能不是Federation，检查格式化时，两个cluster的clusterID是否一致，要是同一个才正确。

cluster01http://10.20.16.217:50070/
http://10.20.16.218:50070/
cluster02http://10.20.16.221:50070/
http://10.20.16.222:50070/

步骤六、 启动集群

1) 启动zkfc

在namenode1、namenode2、datanode3、datanode4执行如下命令启动zkfc进程，观察两个cluster中的NameNode状态是否有变成active的，有active的表示正常。

1

hadoop-daemon.sh start zkfc

2) 启动DataNode

在namenode1执行如下命令启动DataNode，验证进程是否正常。

1

hadoop-daemons.sh start datanode

3) 启动YARN

在namenode1执行如下命令启动YARN，验证进程是否正常。

1

start-yarn.sh

4) 重启集群

到此已经完成了HDFS Federation的搭建，看看能否用通用脚本启动和停止，正常的话是可以完成HDFS Federation的启动和停止的。

1
2
3
4

stop-yarn.sh
stop-dfs.sh
start-dfs.sh
start-yarn.sh

步骤八、 配置viewFS

前面只在core-site.xml配置了逻辑名称，但是物理目录并不存在，所以现在创建/home和/tmp的物理目录，保证逻辑名称能正常使用，这里我将/home和/tmp都放在cluster01的NameSpace。

1
2

hdfs dfs -mkdir /c1/tmp
hdfs dfs -mkdir /c1/home

四、 部署监控

(一) Ganglia 架构

Ganglia 由如下三部分组成，每个组件需要一些依赖包实现其功能。

  • gweb Ganglia Web，基于PHP脚本实现，读取gmetad收集的rrd数据将其可视化、绘制出表格展示。需要发布到支持PHP的WEB容器，例如apache、nginx等，因为要读取gmetad生成的rrd数据文件，因此gweb一般安装在gmetad所在的机器。
  • gmetad Ganglia METAdata Daemon，服务器端的daemon，它通过轮询收集gmond采集的数据，并聚合cluster的各类监控指标信息，然后保存在本地rrdtool的数据库中。
  • gmond Ganglia MONitor Daemon，安装在需要监控的主机，负责收集所在主机的监控数据。

/* This configuration is as close to 2.5.x default behavior as possible
   The values closely match ./gmond/metric.h definitions in 2.5.x */
globals {
  daemonize = yes
  setuid = yes
  user = hadoop
  debug_level = 0
  max_udp_msg_len = 1472
  mute = no
  deaf = no
  allow_extra_data = yes
  host_dmax = 0 /*secs */
  cleanup_threshold = 300 /*secs */
  gexec = no
  send_metadata_interval = 10 /*secs */
}

/*
 * The cluster attributes specified will be used as part of the <CLUSTER>
 * tag that will wrap all hosts collected by this instance.
 */
cluster {
  name = "kxn_cluster"
  owner = "hadoop"
  latlong = "unspecified"
  url = "unspecified"
}

/* The host section describes attributes of the host, like the location */
host {
  location = "unspecified"
}

/* Feel free to specify as many udp_send_channels as you like.  Gmond
   used to only support having a single channel */
udp_send_channel {
  bind_hostname = yes # Highly recommended, soon to be default.
                       # This option tells gmond to use a source address
                       # that resolves to the machine's hostname.  Without
                       # this, the metrics may appear to come from any
                       # interface and the DNS names associated with
                       # those IPs will be used to create the RRDs.
  host = 10.20.16.217
  port = 8649
  ttl = 1
}

/* You can specify as many udp_recv_channels as you like as well. */
udp_recv_channel {
  port = 8649
}

2) 配置gmetad

gmetad负责收集gmond采集到的数据，因此要指定一个gmond作为数据源，数据要存入rrd数据库，因此要指定rrd的存储目录，按如下配置即可。

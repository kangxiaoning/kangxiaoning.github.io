1. 双下划线开始和结尾的object

在python，双下划线开始和结尾的object称为dunder，意思是double underscore.

salt里也有一些这种特殊的object。

1.1 __virtual__

用来检查requirements是否满足。

不满足：可以返回tuple-(False, reason why the module can not be loaded)。

满足： 返回True或者 __virtualname__

1.2 __virtualname__

可以作为 __virtual__的返回值，可以重新定义module的名字，使命令行使用该变量定义的名字作为module名字。

1.3 __salt__

是一个dictionary，包含Minion端所有可用functions的references。

使用举例：

__salt__['cmd.run']('ps -ef | grep httpd')

1.4 __opts__

a copy of the data from the Master/Minion configuration file.

1.5 __grains__

grains data

1.6 __pillar__

pillar data

1.7 __func_alias__

create a map between aliases used from command line and the actual name of the function.

为了在command line使用list/type等这种在python中被视为保留字的名称作为方法名，因此做了一个map来映射真正的方法名。

2. 配置信息

execution module可以从4个地方获取配置

  1. minion配置文件，会解析成 __opts__
  2. grain data，会解析成 __grains__
  3. pillar data ，会解析成 __pillar__
  4. master配置文件，会解析成 __opts__

grains和pillar可以被execution module和state module使用，其它类型的module不行。

config.get()实现了对配置的获取，按如下顺序搜索，先找到先使用。

  1. __opts__(on the Minion)
  2. __grains__
  3. __pillar__
  4. __opts__(on the Master)

例如：user = __salt__['config.get']('mysql', 'root')

模块load顺序： grains –> pillar –> execution modules

因此，不能在grains module里cross-call execution module，会引起循环依赖问题。

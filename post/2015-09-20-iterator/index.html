1. 可迭代对象

指的是支持 iter 的一个对象。如果对象是实际保存的序列，或者可以在迭代工具环境中（如，for 循环）一次产生一个结果的对象，就看做是可迭代的。

>>>

2. 迭代协议

支持迭代协议的对象有一个 next 方法，调用该方法会前进到下一个结果，在对象末尾会引发 StopIteration 。任何这类对象能以 for 循环或其它迭代工具遍历，因为所有迭代工具内部工作机制都是在每次迭代中调用 next ，并且捕捉 StopIteration 异常来确定何时离开。当for 循环开始时，会通过它传给 iter 内置函数，以便从可迭代对象中获得一个迭代器，返回的对象含有需要的 next 方法。

在对象中从左到右扫描的每种工具都使用了迭代协议，例如：列表解析、in、map、sorted、zip、list、enumerate、tuple等。

总结：for 是一种迭代工具，通过内置的 iter 函数运行迭代协议，调用该对象的 next （或者是 next()）方法，而且会捕捉 StopIteration 异常，决定何时停止循环。

Python 3.0 提供了一个内置函数 next ，它会自动调用一个对象的 next 方法。给定一个可迭代对象 X，调用 next(X) 等同于 X.next()，显然 next(X) 更简单。

Python 2.6.6 (r266:84292, Nov 22 2013, 12:16:22)

Type "help", "copyright", "credits" or "license" for more information.

>>> f = open('abc.log')
#文件有自己的迭代方法，因此不需要 iter(f) 返回一个不同的对象
>>> iter(f) is f
True
>>> f.__next__()
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AttributeError: 'file' object has no attribute '__next__'
>>> f.next()
'abc\n'
>>> next(f)
'def\n'
>>> 

#列表及很多其它的内置对象，不是自身的迭代器，因为它们支持多次打开迭代器，对这样的对象，我们必须调用iter来启动迭代
>>> L = [1,2,3]
>>> iter(L) is L
False
>>> L.next()
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AttributeError: 'list' object has no attribute 'next'
>>> I = iter(L)
>>> I.next()
1
>>> next(I)
2
>>>

# python 3.X 中返回可迭代对象（迭代器），这对较大的程序来说很有用，在计算很大的结果列表时，会节约内存并避免暂停

>>> R

>>>
#如果需要列表的话必须使用 list 函数
>>> list(range(5))
